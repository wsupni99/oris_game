## Эпик: Сервер (Даня)

### Блок: Базовая сеть и модель
- [ ] Поднять ServerSocket (порт 8080, конфиг в properties)
- [ ] Реализовать класс Player (id, имя, состояние, socket, потоки)
- [ ] Реализовать GameState (roomId, режим, список игроков, раунд, таймер)
- [ ] Сделать пул потоков ExecutorService для клиентов
- [ ] Реализовать цикл приёма/отправки сообщений клиенту

### Блок: Протокол и сообщения
- [ ] Определить enum MessageType (JOIN, CHAT, DRAW, GUESS, READY, START, TEXT_SUBMIT, ROUND_UPDATE, FINAL_CHAIN, ERROR)
- [ ] Описать формат JSON-сообщений (type, roomId, payload)
- [ ] Написать парсер/обёртку над ObjectInputStream/ObjectOutputStream для JSON
- [ ] Обработчик JOIN/LEAVE (подключение, отключение, рассылка обновлённого списка игроков)
- [ ] Обработчик CHAT (широковещательно в комнату)
- [ ] Обработчик DRAW (ретрансляция мазков всем игрокам комнаты)
- [ ] Обработчик GUESS (сравнение с загаданным словом, отправка CORRECT)
- [ ] Обработчик READY/START (логика готовности и запуска раунда)
- [ ] Обработчик TEXT_SUBMIT/ROUND_UPDATE (логика “глухого телефона”: сдвиги текст/рисунок)

### Блок: Логика режимов
- [ ] Режим 1: Угадай рисунок (создание комнаты, установка слова, проверка угадывания, завершение раунда)
- [ ] Режим 2: Глухой телефон (создание цепочек, хранение текста и рисунков по игрокам и шагам)
- [ ] Реализовать таймер раундов (ScheduledExecutorService)
- [ ] Отправка FINAL_CHAIN с полной последовательностью всем участникам
- [ ] Обработка ошибок протокола (неизвестный тип, некорректные данные)

### Блок: Технический долг и тесты
- [ ] Конфигурируемые порты/тайминги через properties
- [ ] Логирование (уровни INFO/ERROR)
